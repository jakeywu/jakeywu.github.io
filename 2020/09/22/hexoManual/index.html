个人博客WiKi系统搭建

发表于2020-09-22更新于2020-09-23

1. 概念

个人博客(weblog) 字面意思为网络日记，用于记录个人生活，技术等信息，可逐步形成自己的维基百科,从而也可以称为WiKi系统， 一般都是非交互式(静态)网站。

2. 选型

市面上开源的wiki系统有很多，列举部分如下：

  • MediaWiki 复杂且强大
  • DokuWiki 类似MediaWiki，可协同作业，更适搭建合团队知识库
  • minDoc 前身是SmartWiki，国产Go开源,适合搭建团队知识库
  • Gitbook 基于Github编写的系统，有内置站内搜索功能, 看到过很多大佬基于该工具写电子书
  • Hexo 基于nodejs实现的系统， 组件多，功能齐全，速度快

个人只是粗略对比了一下，我个人选择了Hexo. 具体可以自行百度，也可以参考:

  • 如何在语雀，gitbook,看云和hexo个人博客取舍?
  • 个人 Wiki 工具比较（知识管理向）
  • 搭建自己的wiki知识管理系统

3. 安装Hexo

官网文档很贴心，可自行根据其描述搭建, 如下步骤是我自己在Ubuntu18.04.5上面实践总结，各位看官大老爷可忽略接下来的内容。

3.1 环境准备
3.1.1 安装nodejs,最新

访问Nodejs官网下载最新稳定包到指定目录,并解压. 本人目录为/opt/node-v12.18.4-linux-x64，然后执行如下操作

分别执行node -v和npm -v脚本， 均返回版本信息，即安装成功。

3.1.2 安装Git

执行sudo apt-get install git-core脚本即可。

3.2 安装Hexo

执行npm install -g hexo-cli进行安装， 输入hexo -V返回版本信息，即安装成功。

4. 使用Hexo

如上，官网描述很清楚，可根据官网学习使用。 以下是我在实践过程中，认为比较重要的点。

4.1 建站后各种文件的理解
4.1.1 scaffolds文件夹

此文件为模板文件, 包含post.md, draft.md, page.md. 用户可以自行定义模板文件，定义参数见Hexo写作,这部分信息尽量多次阅读理解。

4.1.2 source文件夹

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，其余Markdown和HTML文件会被解析解析并放到 public 文件夹，剩下的文件(如数据，图片等文件)会被拷贝过去。

4.2 主题themes

用户可以根据喜好，从主题库中选择合适的主题(即页面呈现形式),我选择的是Stun, 可在对应的themes目录下找到_config.yml自行配置。
